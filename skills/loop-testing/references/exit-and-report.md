# references/exit-and-report.md — 退出条件与最终报告

## 1. 「收敛低风险轮」单轮判据（全部满足才算一轮收敛）

一轮同时满足以下全部条件，才计作「收敛低风险轮」：

1. **本轮新发现全部为 P3**（0 个也算）；**任何 P0-P2 新发现，即使当轮已修好，本轮也不合格**。
2. 对早先已记录的 P0-P2，本轮没有发现扩大其影响、提高其严重度或推翻既有根因/修复结论的新证据（重复触发同一已知症状不伪装成「新问题」，但仍要关联证据）。
3. 本轮开始前已知但仍未解决的 P0-P2，均已严格归入 `NEEDS_CONFIRMATION / BLOCKED / WONT_FIX`，具备完整复现、影响、尝试记录与明确下一步。
4. **本轮计划场景均已实际执行**，没有用跳过、阻塞或仅看代码代替。
5. 所有未被已知问题明确阻断的核心/高风险旅程均已覆盖，且本轮**至少完成一条端到端冒烟旅程**。
6. 构建 / 测试 / lint / 类型检查 / 健康检查**无新增失败**；既有失败已与对应问题关联且未恶化。
7. **完成全功能回归，实际用例数不得明显低于此前轮次**（防「少测凑零新增」；覆盖明显缩水的轮次不计入连续计数）。

满足 → `converged_streak += 1`；出现 P0-P2 新发现 / 扩大已知重要问题 / 重要回归 / 无理由漏测本轮场景 / 新关键检查失败 → `converged_streak = 0`。

## 2. 正常停止：连续 K=2 轮收敛

- 只有**连续两轮**都是「收敛低风险轮」才可正常停止；
- **第二轮必须使用与第一轮实质不同的场景组合来反证结果**（无新问题的一轮可计入，但不能靠重复同一脚本凑数）。

达标后按真实交付状态二选一：

- **`PASS`**：不存在未解决 P0-P2；所有核心功能为 `PASS` 或合理 `N/A`；所有适用检查通过；无关键证据缺口。
- **`CONVERGED_WITH_OPEN_ISSUES`**：连续两轮只新增 P3 或无新问题，但仍有已充分记录的 P0-P2 / 待确认 / 被阻断功能 / 稳定的既有检查失败。表示「继续盲测的边际价值已很低」，**不表示项目通过**。

## 3. 保险停止（必须如实，禁止谎报 PASS）

- 达 `MAX_ROUNDS`（默认 12）仍未收敛 → **`INCOMPLETE`**；
- 全部剩余有价值工作都被同一权限/环境边界阻塞 → **`BLOCKED`**。

两者都必须先持久化状态，明确下一步与恢复入口。**以下都不是成功停止理由**：已做很多工作、所有已有测试通过、暂时想不到场景、上下文快满、某功能被阻塞、达到最大轮数、本轮没发现问题但覆盖仍不完整。

## 4. 停止前：沙箱清理（`scripts/sandbox-clean.sh`）

输出报告前执行：`bash skills/loop-testing/scripts/sandbox-clean.sh`。它会停掉**自己启动的**进程（`docs/looptesting/.pids` 记录）、移除**自己创建且能确认归属的** worktree/临时数据；**保留 `docs/looptesting/` 全部证据与状态文档，保留 qa 分支（含修复 commit）与基线标记，绝不碰用户数据**。无归属标记时**失败即拒清**（fail-closed）。

## 5. FINAL_REPORT.md 结构（模板见 templates/）

> `FINAL_REPORT.md` **只在退出时**从本技能 `templates/FINAL_REPORT.md` 实例化——启动时不预建（半路存在"最终报告"会误导续跑判断）。写完最终报告、跑完 `sandbox-clean.sh` 后，把 `STATE.md` 的 `status` 写为对应终态；stop-gate 检测到终态即放行并解除哨兵。

1. **FINAL STATUS**：`PASS / CONVERGED_WITH_OPEN_ISSUES / INCOMPLETE / BLOCKED`；
2. 停止原因与证据：总轮数、连续收敛轮数、最后两轮差异与结果、为何不是更高等级；
3. 覆盖摘要：功能总数与 PASS/FAIL/BLOCKED/N-A 数量、核心旅程列表、测试盲区；
4. 已修复问题：ID、级别、根因、回归断言、验证证据、commit 哈希（问题↔commit 对照）；
5. 未解决问题：按 P0→P3，逐条影响/复现/证据/状态/不能修的原因/建议动作；
6. **⚠️ 待用户逐条确认**：每条单列候选方案、你的推荐及各方案影响，**不合并成模糊问题**，对应 `decisions/DEC-xxx.md`；
7. 既有/环境问题：与本轮改动明确区分；
8. 验证清单：实际运行的命令与真实用户场景及结果；未执行项与原因；
9. 代码交付：修改文件、本地提交、未提交变更、最终 `git status`，明确声明**从未 push/merge/release/deploy**；
10. 残余风险、证据目录、准确的续跑步骤。

## 6. 诚实原则（贯穿全程）

未重放复验不标 VERIFIED；复现不了如实标 `CANNOT_REPRODUCE` 并附尝试记录，不编故事；不为尽快收敛而隐瞒或降级问题；不夸大覆盖；`INCOMPLETE` 不得写成 PASS。报告基于实际证据，不用「应该没问题」之类无法核验的结论；对外只记简洁的决策依据、操作、证据与结果，不输出冗长思维过程。
